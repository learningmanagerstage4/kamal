🔓 COOP Fix Bypass — Adobe Acrobat

Vulnerability
The OAuth COOP fix sets Cross-Origin-Opener-Policy: same-origin-allow-popups to sever the opener window reference. However, any request containing ?code= in the URL overrides COOP to unsafe-none, restoring the attacker's full control over the opened window.
Proof — Header Comparison
Normal: acrobat.adobe.com/
✓ COOP: same-origin-allow-popups — protected
Bypass: acrobat.adobe.com/?code=x
✗ COOP: unsafe-none — opener reference fully restored
